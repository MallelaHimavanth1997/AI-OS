# Contributing to AI-OS

AI-OS is built as a local-first, open-source automation platform. Contributions should preserve the system’s modular design, privacy-first posture, and production-quality standards.

## Principles

- Prefer small, testable modules with clear ownership.
- Avoid placeholder code and unfinished stubs.
- Keep dependencies minimal and open source.
- Use dependency injection for agents, services, and integrations.
- Add tests and documentation alongside every meaningful feature.

## Workflow

1. Open an issue or describe the planned change.
2. Implement the smallest production-ready slice.
3. Add or update tests.
4. Update docs when behavior or configuration changes.
5. Validate locally before merging.
